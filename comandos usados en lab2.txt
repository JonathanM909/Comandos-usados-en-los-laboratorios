Comandos usados en la practica 1-
/
sudo apt update → actualiza la lista de paquetes disponibles en los repositorios
sudo apt upgrade -y → actualiza todos los paquetes instalados del sistema automáticamente
cat /etc/apt/sources.list → muestra los repositorios configurados en el sistema
grep -v '^#' /etc/apt/sources.list → muestra solo los repositorios activos sin comentarios
apt search bashtop → busca el programa bashtop en los repositorios
sudo apt install bashtop → instala el programa bashtop en el sistema
bashtop → ejecuta el monitor de recursos del sistema
sudo apt purge bashtop → desinstala bashtop eliminando también sus archivos de configuración
sudo apt autoremove → elimina dependencias que ya no son necesarias/

Comandos usados en la practica 2-
/
crontab -e → abre el editor para programar tareas automáticas
0 23 * * * sudo apt update && sudo apt upgrade -y → programa actualización diaria del sistema a las 11 PM
0 3 * * 0 sudo reboot → programa reinicio automático los domingos a las 3 AM
sudo apt install at → instala la herramienta para programar tareas puntuales
sudo systemctl start atd → inicia el servicio de ejecución de tareas "at"
ls /tmp → muestra el contenido actual de la carpeta temporal
echo "rm -rf /tmp/*" | at now + 1 minute → programa la eliminación del contenido de /tmp en 1 minuto
ls /tmp → verifica el contenido de /tmp después de la ejecución/

Comandos usados en la practica 3-
/
lsblk → muestra los discos y particiones disponibles en el sistema
sudo fdisk /dev/sdb → permite crear o modificar particiones en el disco sdb
n → crea una nueva partición dentro de fdisk
p → define la partición como primaria
1 → asigna el número de partición
ENTER → acepta valores por defecto de tamaño
w → guarda los cambios realizados en fdisk
sudo mkfs.ext4 /dev/sdb1 → formatea la partición en sistema de archivos ext4
mkdir /disco_nuevo → crea un directorio que servirá como punto de montaje
sudo mount /dev/sdb1 /disco_nuevo → monta la partición en el directorio especificado
cd /disco_nuevo → accede al directorio donde está montado el disco
touch onel.txt → crea un archivo vacío con ese nombre
ls → muestra el contenido del directorio actual
cd / → vuelve al directorio raíz
sudo umount /dev/sdb1 → desmonta la partición del sistema
sudo mount /dev/sdb1 /mnt → monta la partición en el directorio /mnt
ls /mnt → muestra el contenido del disco montado en /mnt
